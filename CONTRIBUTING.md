# Contribuciones

## Introducción
Agradecemos tu interés en contribuir a este proyecto. Tu participación es fundamental para mejorar y expandir las funcionalidades del mismo, este es un proyecto de código abierto y valoramos todas las contribuciones de la comunidad pero tambien es un proyecto que buscar realizar una labor social mediante la transparencia gubernamental una labor que nos benecia a todos los mexicanos.

## Cómo contribuir
Si deseas contribuir a este proyecto, por favor sigue los siguientes pasos:

1. Haz un fork del repositorio.
2. Crea una nueva rama para tus cambios:
   ```bash
   git checkout -b nombre-de-tu-rama
   ```
3. Realiza tus cambios y asegúrate de que el código sigue las convenciones del proyecto.
4. Ejecuta las pruebas para asegurarte de que todo funciona correctamente.
5. Haz un commit de tus cambios:
   ```bash
   git commit -m "Descripción de los cambios realizados"
   ```
6. Envía un pull request al repositorio original.

## Como reportar problemas, errores, bugs y otros
Si encuentras algún problema, error o bug en el proyecto, por favor sigue estos pasos para reportarlo:
1. Verifica si el problema ya ha sido reportado buscando en los issues existentes.
2. Si el problema no ha sido reportado, crea un nuevo issue describiendo el problema detalladamente. Incluye información relevante como:
   - Pasos para reproducir el problema.
   - Comportamiento esperado.
   - Comportamiento actual.
   - Capturas de pantalla o registros si es posible.
3. Envía el issue y espera a que los mantenedores del proyecto lo revisen.

## Como proponer nuevas funcionalidades
Si tienes una idea para una nueva funcionalidad o mejora, por favor sigue estos pasos para proponerla:
1. Verifica si la funcionalidad ya ha sido propuesta buscando en los issues existentes.
2. Si la funcionalidad no ha sido propuesta, crea un nuevo issue describiendo la funcionalidad detalladamente. Incluye información relevante como:
   - Descripción de la funcionalidad.
   - Beneficios de la funcionalidad.
   - Cualquier detalle técnico relevante.
3. Envía el issue y espera a que los mantenedores del proyecto lo revisen.

## Requisitos tecnicos o de estilo del código

- Sigue las convenciones de codificación del proyecto.
- Asegúrate de que tu código es compatible con las versiones de PHP y otras dependencias utilizadas en el proyecto.
- Incluye comentarios y documentación en tu código cuando sea necesario.

## Código de conducta
Al contribuir a este proyecto, aceptas seguir el código de conducta del proyecto. Por favor, respeta a los demás miembros de la comunidad y mantén un ambiente colaborativo y positivo.
